## 🚀 第二步：推送代码

打开终端，执行：

```bash
cd /Users/jojodd/my-tool-platform

```

### 方法2：如果推送失败

如果遇到 SSL 证书问题，可以：

**选项A：使用 SSH（如果配置了 SSH key）**
```bash
git remote set-url origin git@github.com:jojodd77/-.git
git push origin main
```

**选项B：使用 GitHub Desktop**
1. 打开 GitHub Desktop
2. 选择仓库
3. 点击 "Push origin"

**选项C：临时禁用 SSL 验证（不推荐，仅测试用）**
```bash
git config --global http.sslVerify false
git push origin main
git config --global http.sslVerify true
```

## 🔍 如果遇到问题

### 问题1：找不到仓库

**解决**：
- 在 GitHub 设置中授权 Vercel 访问
- 点击 "Adjust GitHub App Permissions"

### 问题2：构建失败

**解决**：
- 查看 Vercel 构建日志
- 确保本地 `npm run build` 能成功

### 问题3：API 调用失败

**解决**：
- 检查环境变量是否配置
- 查看 Vercel Function Logs
