## 🎯 部署步骤（按顺序执行）

### 步骤1：推送代码到 GitHub

在终端执行：

```bash
cd /Users/jojodd/my-tool-platform
git push origin main
```

**如果推送失败**（SSL 证书问题），可以：

**方法A：使用 GitHub Desktop**
1. 打开 GitHub Desktop
2. 选择仓库
3. 点击 "Push origin"

**方法B：临时禁用 SSL 验证（仅测试）**
```bash
git config --global http.sslVerify false
git push origin main
git config --global http.sslVerify true
```

**方法C：检查网络连接**
- 确保网络正常
- 或使用 VPN

---

### 步骤2：访问 Vercel

1. 打开浏览器
2. 访问：**https://vercel.com**
3. 点击 "Sign Up" 或 "Log In"
4. 选择 **"Continue with GitHub"**
5. 授权 GitHub 访问

---

### 步骤3：导入项目

1. 登录后，点击 **"+ New Project"** 或 **"Add New Project"**
2. 在 "Import Git Repository" 中找到你的仓库
   - 仓库名：`-` 或显示为 `jojodd77/-`
   - 如果看不到，点击 **"Adjust GitHub App Permissions"** 授权
3. 找到后，点击 **"Import"**

---

### 步骤4：配置项目

Vercel 会自动检测 Next.js，通常不需要修改：

- ✅ Framework Preset: Next.js（自动）
- ✅ Root Directory: `./`（默认）
- ✅ Build Command: `npm run build`（默认）
- ✅ Output Directory: `.next`（默认）

**直接进入下一步配置环境变量！**

---

### 步骤5：配置环境变量（重要！）

在项目配置页面，找到 **"Environment Variables"** 部分：

1. 点击 **"Add"** 或 **"+"** 按钮
2. 添加环境变量：
   - **Key**: `ZHIPU_API_KEY`
   - **Value**: `dff3af0865ac4a75b7511dd22d6188b8.JBxKqerah1FuJVba`
   - **Environment**: 全部勾选
     - ☑ Production
     - ☑ Preview
     - ☑ Development
3. 点击 **"Add"** 保存

**这一步非常重要！没有环境变量，大模型 API 无法使用！**

---

### 步骤6：开始部署

1. 点击 **"Deploy"** 按钮
2. 等待构建完成（通常 2-5 分钟）
3. 看到 **"Congratulations!"** 或 **"Deployment successful"** 表示成功！

---

### 步骤7：获得访问 URL

部署成功后，你会看到：

- ✅ **Production URL**: `https://your-project-name.vercel.app`
- 这个 URL 就是你的公开访问地址！

**复制这个 URL，分享给其他人！** 🎉

## 🔧 常见问题

### Q1: 找不到仓库？

**解决**：
- 在 GitHub 设置中授权 Vercel 访问所有仓库
- 点击 "Adjust GitHub App Permissions"

### Q2: 构建失败？

**解决**：
- 查看 Vercel 的构建日志
- 检查是否有错误信息
- 确保代码没有语法错误

### Q3: API 调用失败？

**解决**：
- 检查环境变量 `ZHIPU_API_KEY` 是否配置
- 在 Vercel 项目设置中查看环境变量
- 查看 Vercel 的 Function Logs

### Q4: 推送代码失败？

**解决**：
- 使用 GitHub Desktop
- 或检查网络连接
- 或使用 SSH 方式
