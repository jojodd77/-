## 🎯 第一步：访问 Vercel

1. **打开浏览器**
2. **访问**：https://vercel.com
3. **点击右上角 "Sign Up"** 或 **"Log In"**

## 🔐 第二步：登录 Vercel

1. **选择登录方式**：点击 **"Continue with GitHub"**
2. **授权 GitHub**：允许 Vercel 访问你的 GitHub 账号
3. **完成登录**

---

## 📦 第三步：导入项目

1. **点击 "+ New Project"** 或 **"Add New Project"** 按钮
2. **在 "Import Git Repository" 中**：
   - 找到你的仓库：`jojodd77/-` 或显示为 `-`
   - 如果看不到，点击 **"Adjust GitHub App Permissions"** 授权访问所有仓库
3. **找到仓库后，点击 "Import"**

---

## ⚙️ 第四步：配置项目

Vercel 会自动检测 Next.js，通常不需要修改：

- ✅ **Framework Preset**: Next.js（自动检测）
- ✅ **Root Directory**: `./`（默认）
- ✅ **Build Command**: `npm run build`（默认）
- ✅ **Output Directory**: `.next`（默认）

**直接进入下一步配置环境变量！**

---

## 🔑 第五步：配置环境变量（重要！）

这一步非常重要！没有环境变量，大模型 API 无法使用。

1. **在项目配置页面，找到 "Environment Variables" 部分**
2. **点击 "Add" 或 "+" 按钮**
3. **添加环境变量**：
   - **Key**: `ZHIPU_API_KEY`
   - **Value**: `dff3af0865ac4a75b7511dd22d6188b8.JBxKqerah1FuJVba`
   - **Environment**: 全部勾选
     - ☑ Production
     - ☑ Preview
     - ☑ Development
4. **点击 "Add" 保存**

**重要**：确保环境变量已添加，否则部署后 API 无法使用！

---

## 🚀 第六步：开始部署

1. **点击 "Deploy" 按钮**（通常在页面底部）
2. **等待构建完成**：
   - 通常需要 2-5 分钟
   - 可以看到构建进度
   - 不要关闭页面

3. **看到 "Congratulations!" 或 "Deployment successful"** 表示成功！

---

## 🔧 如果遇到问题

### 问题1：找不到仓库

**解决**：
- 点击 "Adjust GitHub App Permissions"
- 授权 Vercel 访问所有仓库
- 重新导入项目

### 问题2：构建失败

**解决**：
- 查看 Vercel 的构建日志
- 检查是否有错误信息
- 确保代码没有语法错误

### 问题3：API 调用失败

**解决**：
- 检查环境变量 `ZHIPU_API_KEY` 是否配置
- 在 Vercel 项目设置中查看环境变量
- 查看 Vercel 的 Function Logs
