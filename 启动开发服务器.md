# 🚀 启动开发服务器指南

## ❌ 问题：连接被拒绝 (ERR_CONNECTION_REFUSED)

这个错误表示开发服务器没有运行。需要先启动开发服务器才能访问。

---

## 📋 启动步骤

### 第一步：打开终端

打开终端（Terminal），确保在项目目录：

```bash
cd /Users/jojodd/my-tool-platform
```

### 第二步：安装依赖（如果还没安装）

```bash
npm install
```

这会安装所有必要的依赖包（Next.js、React、TypeScript 等）。

**如果遇到权限问题**，可以尝试：
```bash
sudo npm install
```

或者检查 npm 配置：
```bash
npm config get prefix
```

### 第三步：启动开发服务器

```bash
npm run dev
```

你会看到类似这样的输出：

```
▲ Next.js 14.2.5
- Local:        http://localhost:3001
- Ready in 2.3s
```

### 第四步：访问应用

看到 "Ready" 后，打开浏览器访问：

**http://localhost:3001**

---

## ✅ 验证服务器是否运行

启动成功后，你应该能看到：

1. **终端显示**：
   ```
   ▲ Next.js 14.2.5
   ```

2. **浏览器显示**：

## 🔧 常见问题

### 问题 1：端口 3001 已被占用

**解决方案**：
```bash
# 查看占用端口的进程
lsof -i :3001

# 或者使用其他端口
npm run dev -- -p 3002
```

### 问题 2：npm install 权限错误

**解决方案**：
```bash
# 方法1：使用 sudo（不推荐，但可以临时解决）
sudo npm install

# 方法2：修复 npm 权限（推荐）
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules

# 方法3：使用 nvm 管理 Node.js 版本
```

### 问题 3：依赖安装失败

**解决方案**：
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

### 问题 4：还是看到标注平台

**解决方案**：
1. 确认终端显示的是 `http://localhost:3001`
2. 清除浏览器缓存
3. 使用无痕模式打开
4. 确认访问的是正确的 URL
