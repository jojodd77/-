## 🎯 第二步：部署到 Vercel

### 1. 访问 Vercel

打开浏览器，访问：**https://vercel.com**

### 2. 登录/注册

- 点击 "Sign Up" 或 "Log In"
- 选择 **"Continue with GitHub"**
- 授权 GitHub 访问

### 3. 导入项目

1. 登录后，点击 **"+ New Project"** 或 **"Add New Project"**
2. 在 "Import Git Repository" 中，找到你的仓库
   - 仓库名可能是：`-` 或显示为你的用户名
   - 如果看不到，点击 **"Adjust GitHub App Permissions"** 授权访问所有仓库
3. 找到仓库后，点击 **"Import"**

### 4. 配置项目

Vercel 会自动检测 Next.js，通常不需要修改：

- **Framework Preset**: Next.js（自动检测）✅
- **Root Directory**: `./`（默认）✅
- **Build Command**: `npm run build`（默认）✅
- **Output Directory**: `.next`（默认）✅

**直接点击 "Deploy" 旁边的设置，先配置环境变量！**

### 5. 配置环境变量（重要！）

在部署前，必须配置环境变量：

1. 在项目配置页面，找到 **"Environment Variables"** 部分
2. 点击 **"Add"** 或 **"+"**
3. 添加环境变量：
   - **Key**: `ZHIPU_API_KEY`
   - **Value**: `dff3af0865ac4a75b7511dd22d6188b8.JBxKqerah1FuJVba`
   - **Environment**: 全部勾选
     - ☑ Production
     - ☑ Preview  
     - ☑ Development
4. 点击 **"Add"** 保存

### 6. 开始部署

1. 点击 **"Deploy"** 按钮
2. 等待构建完成（通常 2-5 分钟）
3. 看到 **"Congratulations!"** 表示部署成功！

## 🔧 如果遇到问题

### 问题1：找不到仓库

**解决**：
- 在 GitHub 设置中，确保仓库是公开的，或已授权 Vercel 访问
- 点击 "Adjust GitHub App Permissions" 重新授权

### 问题2：构建失败

**解决**：
- 查看 Vercel 的构建日志
- 检查是否有错误信息
- 确保本地 `npm run build` 能成功

### 问题3：API 调用失败

**解决**：
- 检查环境变量 `ZHIPU_API_KEY` 是否配置
- 在 Vercel 项目设置中查看环境变量
- 查看 Vercel 的 Function Logs
