## 🔍 验证服务器是否运行

### 方法 1：检查端口占用

在终端运行：
```bash
lsof -i :3001
```

如果看到类似以下输出，说明服务器正在运行：
```
COMMAND   PID   USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
node    12345 jojodd   17u  IPv6  ...      0t0  TCP *:3001 (LISTEN)
```

### 方法 2：访问网站

在浏览器中打开：
- **主页**: http://localhost:3001
- **发音修正页面**: http://localhost:3001/correction

### 方法 3：检查进程

在终端运行：
```bash
ps aux | grep "next dev" | grep -v grep
```

如果看到进程，说明服务器正在运行。

## 🐛 如果服务器没有启动

如果页面仍然转圈或无法访问，请：

### 1. 查看终端输出

服务器在后台启动，可能看不到输出。请手动在前台启动：

```bash
cd /Users/jojodd/my-tool-platform

# 停止后台进程
pkill -9 -f "next dev"

# 在前台启动（重要：可以看到错误信息）
npm run dev
```

### 2. 查看错误信息

启动时注意查看终端输出：

**正常启动应该看到：**
```
▲ Next.js 14.2.5
- Local:        http://localhost:3001
- Ready in X.Xs
```

**如果有错误，会显示：**
- 编译错误
- 端口占用错误
- 模块未找到错误
- 配置错误

### 3. 常见问题解决

#### 问题 1：端口仍被占用

```bash
# 强制停止
sudo lsof -ti :3001 | xargs sudo kill -9
sleep 3
npm run dev
```

#### 问题 2：模块未找到

```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
npm run dev
```

#### 问题 3：编译错误

查看终端中的 TypeScript 错误，修复代码后重新启动。
