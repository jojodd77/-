## ⚠️ 豆包 - 模型名称问题

**错误信息**：`404 - The model or endpoint doubao-pro-32k does not exist`

**问题分析**：
- 你的 API Key：`3a80a3f6-4cf0-430a-bb28-274dde2ace11`
- 这不是端点ID格式（端点ID通常是 `ep-` 开头）
- 需要找到正确的模型名称

**解决方法**：

### 方法 1：查看豆包控制台（推荐）

1. 登录豆包控制台：https://console.volcengine.com/
2. 进入"模型服务"或"API 服务"
3. 查看你的模型端点或可用的模型名称
4. 在 `.env.local` 中配置：
   ```bash
   DOUBAO_MODEL=你看到的模型名称
   ```

### 方法 2：尝试不同的模型名称

在 `.env.local` 中尝试：

```bash
# 选项 1
DOUBAO_MODEL=doubao-lite-32k

# 选项 2  
DOUBAO_MODEL=doubao-lite-4k

# 选项 3
DOUBAO_MODEL=doubao-pro-4k
```

### 方法 3：查看 API 文档

访问豆包 API 文档：https://www.volcengine.com/docs/82379
查看正确的模型名称格式。
