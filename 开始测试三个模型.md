# 🚀 开始测试三个模型

## ✅ API Keys 已配置

已为你配置了以下三个模型的 API Keys：

1. **智谱AI** ✅
2. **DeepSeek** ✅  
3. **豆包** ✅

## 📝 开始测试

### 方法 1：通过网页界面测试（推荐）

1. **启动开发服务器**：
   ```bash
   npm run dev
   ```

2. **访问测试页面**：
   打开浏览器访问：**http://localhost:3001/test-models**

3. **运行测试**：
   - 点击 **"开始测试"** 按钮
   - 等待测试完成（可能需要 5-10 分钟）
   - 查看测试结果

### 方法 2：直接调用 API

```bash
curl http://localhost:3001/api/test-models
```

## 📊 测试结果说明

测试完成后会显示：

1. **模型性能汇总表**：
   - 模型名称
   - 准确率（百分比）
   - 平均耗时（毫秒）
   - 通过/总数

2. **详细测试结果**：
   - 每个测试用例的输入、期望、实际结果
   - 是否正确标注
   - 响应时间

## 🎯 评估标准

- **准确率 90%以上**：优秀 ✅
- **准确率 70-90%**：良好 ⚠️
- **准确率 70%以下**：需要改进 ❌

## ⚠️ 注意事项

1. **测试时间**：完整测试可能需要 5-10 分钟，请耐心等待
2. **API 费用**：测试会消耗 API 调用额度
3. **网络连接**：确保网络连接稳定
4. **API 限流**：测试脚本会自动添加延迟，避免触发限流

## 🔧 如果遇到问题

### 问题 1：API Key 未配置
- 检查 `.env.local` 文件是否存在
- 确认 API Keys 是否正确

### 问题 2：某个模型测试失败
- 检查该模型的 API Key 是否有效
- 检查是否有足够的 API 额度
- 查看错误信息，可能是 API 端点或格式问题

### 问题 3：豆包 API 错误
- 豆包 API 可能需要特定的模型端点 ID
- 如果失败，可以在 `.env.local` 中添加 `DOUBAO_MODEL=你的模型端点ID`

## 📈 测试完成后

根据测试结果：
1. 选择准确率最高的模型
2. 如果准确率相近，选择响应时间最短的
3. 考虑 API 调用成本

然后可以在 `app/api/correct/route.ts` 中更新默认使用的模型。
