# 发音修正平台 - 项目需求

## 📋 项目概述

一个发音修正平台，选用一个大模型接入，用 prompt 把已有的规则告诉他，用户输入文本后，根据规则判断用户的文本是否符合规则，符合则输出：符合规则，不符合规则就修改为符合规则的文本再输出。

---

## 🎯 主要功能

### 1. 发音判断修正

**功能描述**：
- 用户输入文本
- 系统判断文本是否符合发音规则
- 符合规则：输出"符合规则"
- 不符合规则：自动修改为符合规则的文本并输出

**技术实现**：
- 接入大模型 API（如 OpenAI、Claude、文心一言等）
- 使用 Prompt 工程，将发音规则传递给大模型
- 前端输入框 + 后端 API 处理

### 2. 历史记录

**功能描述**：
- 记录用户的输入历史
- 可以查看历史记录
- 显示：原始文本、修正后文本、时间戳等

**技术实现**：
- 数据库存储（PostgreSQL/MySQL/Supabase）
- 用户会话管理
- 历史记录列表页面

### 3. 规则查看

**功能描述**：
- 提供规则文档页面
- 用户可以查看所有发音规则
- 规则说明清晰易懂

**技术实现**：
- 规则文档页面
- 可以是静态文档或从数据库读取

---

## 🛠️ 技术栈

- **前端**：Next.js 14 + TypeScript + Tailwind CSS
- **后端**：Next.js API Routes
- **数据库**：PostgreSQL / Supabase（推荐）
- **AI 模型**：大模型 API（待选择）
