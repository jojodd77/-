## 🚀 部署步骤

### 第一步：提交代码到 GitHub

如果还有未提交的更改：

```bash
cd /Users/jojodd/my-tool-platform
git add .
git commit -m "准备部署到 Vercel"
git push
```

### 第二步：访问 Vercel

1. 打开浏览器，访问：https://vercel.com
2. 点击 "Sign Up" 或 "Log In"
3. 选择 "Continue with GitHub"
4. 授权 GitHub 访问

### 第三步：导入项目

1. 登录后，点击 "Add New Project" 或 "+ New Project"
2. 在 "Import Git Repository" 中，选择你的仓库
   - 如果看不到，点击 "Adjust GitHub App Permissions" 授权
3. 选择仓库后，点击 "Import"

### 第四步：配置项目

Vercel 会自动检测 Next.js，通常不需要修改：

- **Framework Preset**: Next.js（自动）
- **Root Directory**: `./`（默认）
- **Build Command**: `npm run build`（默认）
- **Output Directory**: `.next`（默认）

### 第五步：配置环境变量（重要！）

在 "Environment Variables" 部分：

1. 点击 "Add" 或 "+"
2. 添加：
   - **Key**: `ZHIPU_API_KEY`
   - **Value**: `dff3af0865ac4a75b7511dd22d6188b8.JBxKqerah1FuJVba`
   - **Environment**: 全部勾选（Production, Preview, Development）

3. 点击 "Add" 保存

### 第六步：部署

1. 点击 "Deploy" 按钮
2. 等待构建完成（通常 2-5 分钟）
3. 看到 "Congratulations!" 表示部署成功

### 第七步：获得 URL

部署成功后，你会看到：
- ✅ **Production URL**: `https://your-project.vercel.app`
- 这个 URL 就是你的公开访问地址！

## ✅ 部署后验证

1. **访问 URL**：打开浏览器访问你的 URL
2. **测试功能**：
   - 进入"发音修正"页面
   - 输入测试文本：`图中哪个光路图是正确的？`
   - 点击"判断并修正"
   - 应该能看到修正结果

3. **检查日志**：
   - 在 Vercel 项目页面，点击 "Functions" 或 "Logs"
   - 查看是否有错误

---

## 🔧 如果部署失败

### 常见问题

1. **构建失败**：
   - 查看 Vercel 的构建日志
   - 检查是否有语法错误
   - 确保 `npm run build` 在本地能成功

2. **API 调用失败**：
   - 检查环境变量是否配置
   - 检查 API Key 是否有效
   - 查看 Vercel 的 Function Logs

3. **找不到仓库**：
   - 检查 GitHub 仓库是否公开或已授权
   - 在 GitHub 设置中授权 Vercel 访问
