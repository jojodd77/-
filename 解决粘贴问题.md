# 🔐 解决终端无法粘贴问题

## ⚠️ 为什么无法粘贴？

这是**正常的安全机制**！终端在输入密码时会禁用粘贴，防止密码泄露。

---

## ✅ 解决方法

### 方法1：手动输入 Token（推荐）

1. **获取 Token**：
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token" → "Generate new token (classic)"
   - 填写信息，勾选 `repo`
   - 生成并复制 Token

2. **手动输入**：
   - 在终端中，**手动输入** Token（虽然不能粘贴，但可以手动输入）
   - Token 通常很长，但可以分段输入
   - 输入时不会显示字符（这是正常的）

3. **按回车**完成

---

### 方法2：使用 GitHub Desktop（最简单）⭐

**推荐使用这个方法，无需手动输入！**

1. **打开 GitHub Desktop**
2. **选择仓库**：`my-tool-platform`
3. **点击 "Push origin"** 按钮
4. **完成！** 会自动处理认证

---

### 方法3：配置 Git Credential Helper（避免每次输入）

配置后，只需要输入一次，之后会自动保存：

```bash
# 配置 credential helper
git config --global credential.helper osxkeychain

# 然后推送（会要求输入一次，之后自动保存）
git push origin main
```

这样配置后，Token 会保存在 macOS 的钥匙串中，以后就不需要再输入了。

---

## 🎯 推荐方案

**使用 GitHub Desktop**，最简单，无需手动输入 Token！

---

## 📝 如果选择手动输入 Token

Token 通常格式类似：
```
ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

虽然不能粘贴，但可以：
1. 打开 Token 页面
2. 看着 Token
3. 在终端中手动输入
4. 输入时不会显示字符（正常的安全机制）

---

## 💡 提示

- 终端输入密码时，**不会显示字符**（包括 Token），这是正常的
- 输入完成后按回车即可
- 如果输入错误，可以按 `Ctrl + C` 取消，重新开始

---

**推荐使用 GitHub Desktop，最简单！** 🚀
