# 🔗 配置应用链接

## 📝 说明

`http://localhost:3001` 是**开发环境**的本地地址，用于本地开发测试。

**生产环境**（部署后）需要使用实际的域名。

---

## 🔧 配置步骤

### 1. 开发环境（本地）

开发时使用 `http://localhost:3001` 是正常的，这是本地开发服务器。

如果需要修改端口：

```bash
# 使用自定义端口（例如 3001）
npm run dev -- -p 3001

# 或修改 package.json 中的脚本
```

### 2. 生产环境（部署后）

部署后需要配置实际的生产环境 URL。

#### 方法一：环境变量配置

创建 `.env.local` 或 `.env.production`：

```bash
# 生产环境
NEXT_PUBLIC_APP_URL=https://your-platform.com
```

#### 方法二：在代码中使用

```typescript
// lib/config.ts
export const APP_URL = process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3001';
```

---

## 🌐 常见部署平台配置

### Vercel

1. 在 Vercel 项目设置中添加环境变量：
   - `NEXT_PUBLIC_APP_URL` = `https://your-project.vercel.app`

2. 或使用自定义域名：
   - `NEXT_PUBLIC_APP_URL` = `https://your-domain.com`

### Netlify

在 Netlify 项目设置 → Environment variables 中添加：
- `NEXT_PUBLIC_APP_URL` = `https://your-project.netlify.app`

### 自建服务器

在服务器环境变量或 `.env.production` 中配置：
- `NEXT_PUBLIC_APP_URL` = `https://your-domain.com`
